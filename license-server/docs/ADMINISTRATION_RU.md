# Руководство администратора

В этом руководстве объясняется, как управлять сервером лицензирования RheoLab с помощью веб-панели администратора.

## Доступ к панели администратора

1.  Откройте веб-браузер.
2.  Перейдите по адресу `https://<YOUR_DOMAIN>/admin/`, если настроен HTTPS, либо `http://<IP_ВАШЕГО_СЕРВЕРА>/admin/` для локального/временного доступа.
3.  Войдите, используя учетные данные администратора.
    -   **Пользователь по умолчанию**: `admin`, если не переопределён `RHEOLAB_ADMIN_USER`
    -   **Пароль**:
        при автоматической установке генерируется `install.sh` и сохраняется в `/root/.rheolab-credentials`;
        при ручной установке проверяется bcrypt-хеш `ADMIN_PASS_HASH`, а не plaintext-параметр `ADMIN_PASS`

## Обзор панели управления

На главной странице отображается список всех лицензионных ключей.
- **Статусы**:
    - 🟢 **Active (Активен)**: Лицензия действительна и активна.
    - 🔴 **Expired (Истёк)**: Срок действия лицензии истёк.
    - ⚫ **Revoked (Отозван)**: Лицензия была заблокирована вручную.
    - ⚪ **Inactive (Неактивен)**: Лицензия создана, но еще не активирована пользователем.

## Управление лицензиями

### Создание новой лицензии
1.  Нажмите **"Создать лицензию"**.
2.  Заполните форму:
    - **Customer Name (Имя клиента)**: Название компании или имя пользователя.
    - **Email**: Контактный email (необязательно).
    - **Type (Тип)**:
        - `Trial`: Срок действия 30 дней.
        - `Standard`: базовая платная лицензия.
        - `Developer`: лицензия с developer-функциями desktop-приложения.
        - `Enterprise`: расширенный корпоративный tier.
    - **Expiration Date (Дата истечения)**: Рассчитывается автоматически, но можно изменить.
    - **Max Activations (Макс. активаций)**: Обычно 1 (на один компьютер).
3.  Нажмите **"Create"**.
4.  Скопируйте сгенерированный **Лицензионный ключ** (например, `ABCD-1234-EFGH-5678`) и отправьте его пользователю.

### Редактирование лицензии
1.  Найдите лицензию в списке.
2.  Нажмите иконку **Редактировать** (карандаш).
3.  Вы можете изменить:
    - Данные клиента.
    - Дату истечения (для продления лицензии).
    - Макс. активаций.
    - Заметки.
4.  Нажмите **"Save"**.

### Отзыв лицензии (Блокировка)
Чтобы немедленно заблокировать доступ пользователю:
1.  Найдите лицензию.
2.  Нажмите иконку **Revoke** (блок/запрет).
3.  Подтвердите действие.
Приложение перестанет работать на компьютере пользователя при следующей проверке (обычно при запуске).

### Удаление лицензии
**Внимание**: Это безвозвратно удаляет лицензию и историю её активаций.
1.  Найдите лицензию.
2.  Нажмите иконку **Delete** (корзина).

## Просмотр логов

Вся история активаций и проверок сохраняется в базе данных в таблице `activation_log`. В текущей версии админ-панели просмотр логов доступен через базу данных.

## Устранение неполадок

### Ошибка "Invalid License" у клиента
- Проверьте правильность ввода ключа.
- Проверьте, не истек ли срок действия лицензии.
- Проверьте, не активирована ли лицензия на другом компьютере (несовпадение Machine ID).
    - *Решение*: используйте штатное действие сброса привязки в admin panel; прямое редактирование БД больше не должно быть основным операционным сценарием.

### Ошибки сервера
- Проверьте логи Apache: `/var/log/apache2/license-error.log`
- Проверьте логи PHP.

## Демо-пользователи

Панель администратора автоматически отображает список демо-пользователей (если таблица `demo_users` существует в базе данных).

### Информация в панели демо-пользователей:
- **Machine ID**: Уникальный идентификатор устройства
- **Первый запуск**: Дата первого использования демо-версии
- **Последний визит**: Когда пользователь последний раз открывал приложение
- **IP адрес**: IP-адрес пользователя
- **Дней в демо**: Сколько дней пользователь использует демо (лимит 30 дней)

### Статистика:
- **Всего**: Общее количество демо-пользователей
- **Активных за 30 дн.**: Пользователи, которые заходили в последние 30 дней
- **Новых за 7 дн.**: Пользователи, которые впервые запустили приложение за последние 7 дней

### Примечание
Для работы функционала демо-пользователей необходимо применить миграцию:
```sql
-- migrations/20251231_add_demo_and_discovery.sql
CREATE TABLE IF NOT EXISTS demo_users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    machine_id VARCHAR(64) UNIQUE NOT NULL,
    first_seen_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    last_seen_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    ip_address VARCHAR(45),
    INDEX idx_machine_id (machine_id)
) ENGINE=InnoDB;
```

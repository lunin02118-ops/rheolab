# Руководство пользователя (Менеджер лицензий)

Это руководство предназначено для Менеджера лицензий, который занимается выдачей ключей клиентам.

## Рабочий процесс

1.  **Покупка клиентом**: Клиент приобретает корпоративную лицензию RheoLab.
2.  **Генерация ключа**:
    - Войдите в Панель администратора.
    - Нажмите **"Создать лицензию"**.
    - Введите имя клиента.
    - Выберите тип `Corporate`, `Developer` или `Superuser` в зависимости от сценария выдачи.
    - Нажмите **Create**.
    - Скопируйте сгенерированный ключ (например, `ABCD-1234-EFGH-5678`).
3.  **Отправка ключа**: Отправьте ключ клиенту по email.
4.  **Активация**:
    - Клиент вводит ключ в приложении RheoLab.
    - Приложение соединяется с сервером и активируется.
    - Вы увидите изменение статуса на "Active" и появление Machine ID в панели.

## Частые задачи

### Продление лицензии
Если клиент продлевает подписку:
1.  Найдите его лицензию в списке.
2.  Нажмите иконку **Редактировать** (карандаш).
3.  Измените **Expiration Date (Дата истечения)** на новую дату (например, добавьте 1 год).
4.  Нажмите **Save**.
Приложение клиента автоматически получит новую дату при следующей проверке. Им *не нужно* вводить ключ заново.

### Перенос лицензии (Смена компьютера)
Если клиент купил новый ПК или переустановил Windows:
1.  Они не смогут активировать тот же ключ на новом ПК, так как он привязан к старому Machine ID.
2.  **Процедура**:
    - Убедитесь, что это тот же клиент.
    - Сначала используйте штатный reset-binding в admin panel.
    - Новый ключ выдавайте только если этого требуют бизнес-правила или лимиты активаций.

### Работа с пробными версиями (Trial)
- Выдайте лицензию типа "Trial".
- Она истечет автоматически через 30 дней.
- Если клиент решил купить:
    1.  Найдите пробную лицензию.
    2.  Отредактируйте её.
    3.  Смените **Type** на нужный tier (`Corporate`, `Developer` или `Superuser`).
    4.  Обновите **Expiration Date**.
    5.  Сохраните.
    Клиент продолжит использовать тот же ключ.

# Руководство по миграции сервера лицензирования

Это руководство описывает процесс переноса сервера лицензирования на новый VPS без потери данных и с сохранением активаций пользователей.

## ⚠️ КРИТИЧЕСКИ ВАЖНО: IP-адрес vs Домен

В текущей версии приложения используется домен **`license.vizbuka.ru`**. Это **правильный подход**.

**Преимущество:**
При переезде на новый сервер вам **НЕ нужно** обновлять приложение у клиентов.
Достаточно просто изменить IP-адрес в настройках DNS домена `license.vizbuka.ru`.

**Проблема (если бы был IP):**
Если бы в коде был жестко прописан IP, то при смене сервера все активации слетели бы. Но мы это исправили.

---

## Процесс миграции (Пошагово)

Предполагаем, что вы уже используете доменное имя или готовы обновить приложение у клиентов.

### Шаг 1: Подготовка старого сервера (Бэкап)

1.  Зайдите на текущий сервер по SSH.
2.  Создайте свежую резервную копию:
    ```bash
    /usr/local/bin/backup-license.sh
    ```
3.  Найдите созданный архив:
    ```bash
    ls -lh /var/backups/license-server/
    # Пример: backup_2025-12-29_10-00-00.tar.gz
    ```
4.  Скачайте этот архив на свой компьютер:
    ```bash
    scp root@<OLD_IP>:/var/backups/license-server/backup_2025-12-29_10-00-00.tar.gz ./
    ```

### Шаг 2: Подготовка нового сервера

1.  Закажите новый VPS (Ubuntu 24.04).
2.  Установите ПО сервера лицензирования (см. [INSTALLATION_RU.md](INSTALLATION_RU.md)).
    - Используйте `install.sh` для быстрой настройки.
    - **Важно**: Пароли базы данных могут отличаться от старых, это нормально. Скрипт восстановления это учтет (или вам придется поправить конфиг).

### Шаг 3: Перенос данных (Восстановление)

1.  Загрузите файл бэкапа и скрипт восстановления на новый сервер:
    ```bash
    scp backup_2025-12-29_10-00-00.tar.gz root@<NEW_IP>:/root/
    scp license-server/restore.sh root@<NEW_IP>:/usr/local/bin/restore-license.sh
    ```
2.  Зайдите на новый сервер:
    ```bash
    ssh root@<NEW_IP>
    chmod +x /usr/local/bin/restore-license.sh
    ```
3.  Запустите восстановление:
    ```bash
    /usr/local/bin/restore-license.sh /root/backup_2025-12-29_10-00-00.tar.gz
    ```
    *Скрипт восстановит базу данных (со всеми ключами и привязками Machine ID) и файлы веб-сервера.*

### Шаг 4: Проверка конфигурации

После восстановления файл `config.php` будет заменен файлом со старого сервера.
**Если пароли к БД на новом сервер отличаются**, сайт выдаст ошибку "Database Error".

1.  Откройте конфиг:
    ```bash
    nano /var/www/license-server/config.php
    ```
2.  Проверьте `DB_USER` и `DB_PASS`. Они должны соответствовать MySQL на **новом** сервере.
    - Если вы использовали `install.sh` на новом сервере, посмотрите пароли в его выводе (или сбросьте пароль пользователя `license_user` в MySQL, если забыли).
3.  **LICENSE_SECRET**: Оставьте как есть (из бэкапа). Он должен совпадать со старым, чтобы подписи лицензий оставались валидными.

### Шаг 5: Переключение DNS (Момент истины)

1.  Зайдите в панель управления доменом (регистратор домена).
2.  Измените **A-запись** для `license.rheolab.com` (или вашего домена).
3.  Укажите **IP-адрес нового сервера**.
4.  Сохраните.

### Шаг 6: Проверка

1.  Подождите обновления DNS (от 5 минут до 24 часов).
2.  Попробуйте активировать лицензию или проверить статус в приложении.
    - Приложение должно соединиться с новым сервером.
    - Так как база данных восстановлена, старые ключи будут опознаны.
    - Так как Machine ID клиентов хранятся в БД, привязка к оборудованию сохранится.

## Что делать, если мы использовали IP-адрес?

Если вы не использовали домен, а прописали IP в коде:
1.  Выполните Шаги 1-4 (перенос сервера).
2.  В коде приложения (`src-tauri/src/commands/licensing/types.rs`) измените `LICENSE_SERVER_URL` на **новый IP** (или лучше сразу на домен).
3.  Соберите новую версию приложения.
4.  Разошлите обновление всем клиентам.
    - *Пока клиент не обновит приложение, его лицензия не будет проверяться (ошибка сети).*

# Руководство администратора

Этот документ объясняет, как управлять RheoLab License Server через web-based Admin Panel.

## Доступ к admin panel

1. Откройте браузер.
2. Перейдите на `https://<YOUR_DOMAIN>/admin/`, если настроен HTTPS, или на `http://<YOUR_SERVER_IP>/admin/` для локального/admin-only доступа.
3. Войдите под учётными данными администратора.
   - **Пользователь по умолчанию**: `admin`, если не переопределён `RHEOLAB_ADMIN_USER`
   - **Пароль**:
     в автоматическом сценарии он генерируется installer-ом и сохраняется в `/root/.rheolab-credentials`;
     при ручной установке аутентификация идёт через `ADMIN_PASS_HASH`, plaintext `ADMIN_PASS` больше не является поддерживаемой конфигурацией

## Обзор dashboard

На dashboard отображается список всех лицензионных ключей.

- **Индикаторы состояния**:
  - 🟢 **Active**: лицензия действительна и активна
  - 🔴 **Expired**: срок действия лицензии истёк
  - ⚫ **Revoked**: лицензия вручную отозвана
  - ⚪ **Inactive**: лицензия создана, но ещё ни разу не активирована

## Управление лицензиями

### Создание новой лицензии

1. Нажмите **«Создать лицензию»**.
2. Заполните форму:
   - **Customer Name**: имя клиента или компании
   - **Email**: контактный email, если нужен
   - **Type**:
     - `Trial`: срок действия 30 дней
     - `Corporate`: постоянная корпоративная лицензия с привязкой к железу
     - `Developer`: открывает developer-oriented функции в desktop-клиенте
     - `Superuser`: персональный alpha-tier владельца проекта
   - **Expiration Date**: для `Trial`, `Developer`, `Superuser`; `Corporate` бессрочная
   - **Max Activations**: серверная модель привязывает ключ к одной машине
3. Нажмите **Create**.
4. Скопируйте сгенерированный **License Key** и передайте пользователю.

### Редактирование лицензии

1. Найдите лицензию в списке.
2. Нажмите иконку **Edit**.
3. Можно изменить:
   - данные клиента
   - дату окончания
   - количество активаций
   - заметки
4. Нажмите **Save**.

### Отзыв лицензии

Чтобы немедленно заблокировать пользователя:

1. Найдите лицензию.
2. Нажмите иконку **Revoke**.
3. Подтвердите действие.

На следующей проверке лицензии приложение на машине пользователя перейдёт в неактивное состояние.

### Удаление лицензии

**Внимание:** это безвозвратно удаляет лицензию и её историю.

1. Найдите лицензию.
2. Нажмите иконку **Delete**.

## Просмотр логов

Если вкладка логов доступна в UI, используйте её. В противном случае история активаций и валидаций хранится в таблице `activation_log`.

Текущая admin panel сфокусирована в первую очередь на License Management, а подробные логи лежат в базе данных.

## Устранение неполадок

### Ошибка «Invalid License» у клиента

- Проверьте правильность ключа.
- Проверьте, не истёк ли срок действия лицензии.
- Проверьте, не активирована ли лицензия уже на другом устройстве, из-за чего возник Machine ID mismatch.
  - Решение: используйте штатный reset binding в admin panel, прежде чем выпускать новый ключ.

### Ошибки сервера

- Проверьте Apache logs: `/var/log/apache2/license-error.log`
- Проверьте PHP logs

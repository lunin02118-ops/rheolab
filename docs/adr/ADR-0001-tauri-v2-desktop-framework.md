# ADR-0001: Tauri v2 как фреймворк десктопного приложения

**Статус:** ✅ Реализовано  
**Дата принятия:** до 2026-02-01 (предшествует этому репозиторию)  
**Дата документирования:** 2026-02-27 (ретроспективная запись)  
**Авторы:** Platform Team  
**Затронутые компоненты:** `src-tauri/`, `src/`, `package.json`

---

## 1. Контекст

Необходима десктопная платформа для офлайн-приложения анализа реологии (Windows). Требования:
- Встроенная SQLite без внешних зависимостей
- Нативный Rust-бэкенд для расчётов
- Веб-фронтенд (React) без облачной зависимости
- Одиночный установочный файл для Windows (NSIS)

## 2. Рассмотренные альтернативы

| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| **Tauri v2** | Малый размер бандла (~8 MB), нативный Rust, WebView2 | WebView2 требует Windows 10+, ограниченный Linux-рынок HMI |
| Electron | Кросс-платформа, большая экосистема | ~150 MB поставка, высокий RAM, V8 для всего |
| Qt (PyQt/C++) | Нативный UI, нет WebView overhead | Нет React, дорогая лицензия, низкая скорость разработки |

## 3. Решение

Выбран **Tauri v2** (стабильный релиз 2024-10):
- Rust-бэкенд (`src-tauri/`) — базы данных, расчёты, лицензирование, отчёты
- Vite 6 + React 19 во фронтенде, встроенный в WebView2
- `rusqlite` (bundled) — SQLite без системной зависимости
- IPC через `invoke()` — типизированный мост TS↔Rust через 94 команды

## 4. Последствия

- **Положительные:** бандл ~30 MB (NSIS), <350 MB RAM p95, нативные расчёты без WASM
- **Отрицательные:** привязка к WebView2 (Windows), нет браузерного запуска
- **Нейтральные:** обновления через Tauri Updater (Ed25519-подпись)

# ADR-0002: SQLite + rusqlite как встроенная БД

**Статус:** ✅ Реализовано  
**Дата принятия:** до 2026-02-01 (предшествует этому репозиторию)  
**Дата документирования:** 2026-02-27 (ретроспективная запись)  
**Авторы:** Platform Team  
**Затронутые компоненты:** `src-tauri/src/db/`, `src-tauri/Cargo.toml`

---

## 1. Контекст

Приложение реологии офлайн-first. Данные экспериментов включают временны́е ряды по 20 000+ точек на сессию. Нужна локальная БД без сетевой зависимости, совместимая с Rust.

## 2. Рассмотренные альтернативы

| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| **SQLite + rusqlite (bundled)** | Нулевая системная зависимость, WAL, FTS5, 1 файл | Нет конкурентных записей (решается пулом r2d2) |
| PostgreSQL / MySQL | Конкурентность | Требует отдельного сервера, не подходит для desktop |
| sled (embedded Rust) | Нативный Rust | Нет SQL, нет FTS5, менее зрелая экосистема |
| JSON-файлы | Просто | Нет транзакций, медленный поиск, сложная миграция |

## 3. Решение

Выбрана **SQLite via `rusqlite` с флагом `bundled`**:
- SQLite компилируется в бинарник — нет зависимости от системной библиотеки
- WAL-режим (`PRAGMA journal_mode=WAL`) + `synchronous=NORMAL` — оптимальный баланс надёжности и скорости
- r2d2 connection pool (`db/pool.rs`) — конкурентный доступ из нескольких команд
- FTS5 virtual table для полнотекстового поиска экспериментов
- `SCHEMA_VERSION = 4` (V1_DDL + incremental migrations V2–V4)

## 4. Последствия

- **Положительные:** нет внешних зависимостей при установке, атомарные транзакции, крейт `rusqlite 0.31 bundled` (~3 MB в бинарнике)
- **Отрицательные:** один файл БД — стандартное ограничение SQLite на конкурентную запись (обходится через пул + WAL)
- **Нейтральные:** бэкапы — простое копирование `.db` файла (`commands/backup.rs`)

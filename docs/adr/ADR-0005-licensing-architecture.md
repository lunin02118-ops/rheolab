# ADR-0005: Архитектура лицензирования — двухуровневая криптографическая защита

**Статус:** ✅ Реализовано  
**Дата принятия:** до 2026-02-01 (предшествует репозиторию)  
**Дата документирования:** 2026-04-18 (ретроспективная запись)  
**Авторы:** Platform Team  
**Затронутые компоненты:** `src-tauri/src/commands/licensing/`, `license-server/api/`

---

## 1. Контекст

Десктопное приложение работает офлайн. Лицензии хранятся в локальной SQLite, к которой пользователь имеет физический доступ. Нужна защита от:
- Подмены/клонирования лицензионных данных в БД
- Перенос лицензии между машинами через копирование файлов
- Работы с отозванной лицензией при длительном офлайне

## 2. Рассмотренные альтернативы

| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| **Двухуровневая: HMAC + RSA** | Локальная защита от tamper + серверное доказательство | Сложность реализации, два набора ключей |
| Только HMAC | Проще, один ключ | Ключ зашит в бинарник — извлекается reverse engineering |
| Только RSA (онлайн) | Самая надёжная при наличии сети | Не работает офлайн без кэша |
| Лицензионный dongle | Физическая защита | Неприемлемо для целевого рынка лабораторий |

## 3. Решение

Двухуровневая схема F-07:

1. **HMAC-SHA256 (Level 1)** — защита целостности данных в SQLite.
   - Ключ: `INTEGRITY_SECRET_KEY` (compile-time embedded, runtime override).
   - Сравнение через `subtle::ConstantTimeEq` (защита от timing-атак).

2. **RSA-2048-SHA256 (Level 2)** — серверное доказательство подлинности.
   - Приватный ключ на `license-server/`, публичный — `src-tauri/keys/`.
   - Каждая лицензия подписана сервером; клиент верифицирует при загрузке.
   - Grace-период закрыт (S-2): HMAC-only записи без RSA-подписи отклоняются.

**Machine fingerprint v2:** `SHA256("rheolab-hw-v2-" + cpuId + "|" + moboUUID + "|" + biosSerial)` — три аппаратных компонента из firmware/silicon.

**Офлайн-политика:** 7-дневный кэш онлайн-проверки (AES-256-GCM encrypted). По истечении 30 дней без сети — лицензия блокируется (fail-closed).

## 4. Последствия

- **Положительные:** работа офлайн до 30 дней, защита от клонирования БД, ротация ключей без пересборки
- **Отрицательные:** дополнительная сложность (два уровня crypto, миграция v1→v2 fingerprint)
- **Нейтральные:** demo-режим (30 дней / 10 экспериментов) использует ту же HMAC-инфраструктуру

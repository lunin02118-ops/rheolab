# Альфа-ревью Rheolab — сводный отчёт по найденным багам

Документ агрегирует все замечания, выявленные в ходе обзора кода перед альфа-стэмпом. Порядок — по приоритету.

---

## P1 — блокеры / критические

### 1. `.db` в FS-scope allowlist
**Где:** `src-tauri/capabilities/default.json` (FS scope).
**Проблема:** В allow-листе присутствует маска `*.db`, что даёт фронту прямой доступ к файлам БД в обход бизнес-логики. Для открытия/сохранения достаточно `dialog` plugin — он сам выдаёт разовые пермишены на выбранный путь.
**Действие:** Убрать `*.db` из FS scope. Работу с БД вести исключительно через Rust-команды; выбор файла — через `dialog`.

### 2. Тихий fallback в `get_update_channel`
**Где:** `src-tauri/src/updater.rs`, функция `get_update_channel`.
**Проблема:** При невалидном/отсутствующем канале функция молча возвращает `stable`. В проде это маскирует мисконфиг и даёт иллюзию, что alpha/beta подписан, хотя пользователь сидит на stable-манифесте.
**Действие:** Логировать `logger.warn!` с причиной fallback'а и исходным значением. В dev-сборке — возвращать `Err` при явном мисконфиге.

### 3. Гонка между cancel и prune в `JobScheduler`
**Где:** `src-tauri/src/jobs/scheduler.rs` (взятие `jobs_lock` и `metrics_lock` раздельно).
**Проблема:** `cancel_job` берёт `jobs_lock`, отпускает, затем берёт `metrics_lock`. Параллельный `prune_completed` между этими точками может удалить метрику; cancel завершится на уже «чужом» состоянии. Результат — рассинхрон счётчиков и потенциальная паника в debug-сборке.
**Действие:** Захватывать оба лока в одной критической секции (в фиксированном порядке jobs → metrics) либо перевести на единый `Mutex<SchedulerState>`.

## P2 — важные, но не блокеры

### 4. Нет документации порядка захвата локов
**Где:** над структурой `JobScheduler` в `scheduler.rs`.
**Проблема:** Инвариант «jobs перед metrics» нигде не зафиксирован; будущая правка легко создаст дедлок.
**Действие:** Добавить doc-comment с явным описанием порядка и запретом обратного захвата.

### 5. Sentinel `u64::MAX` для `output_bytes` в `JobMetricState`
**Где:** `scheduler.rs:28, 153, 157–164`.
**Проблема:** Магическое значение `u64::MAX` означает «нет значения». Если реальный размер когда-нибудь достигнет этого числа — превратится в `None` без предупреждения.
**Действие:** Заменить на `AtomicU64 + AtomicBool` (флаг наличия) либо на `Mutex<Option<u64>>`.

### 6. Остаточные `console.*` в production-путях
**Где:**
- `src/lib/parsing/client.ts:102, 110`
- `src/lib/tauri/core.ts:195`
- `src/lib/reports/comparison-experiment-adapter.ts:335`

(Случаи в `converters.ts:169, 231` — внутри JSDoc-примеров, безопасны.)
**Проблема:** Пропущены при миграции на `logger`. Без ESLint-правила регрессия неизбежна.
**Действие:** Заменить на `logger.*` и включить ESLint-rule `no-console` с allowlist для `src/lib/logger.ts`.

### 7. HMAC channel token: ключи зашиты в бинарь
**Где:** `updater.rs:495–506` — `BETA_CHANNEL_KEY`, `ALPHA_CHANNEL_KEY`.
**Проблема:** Статические константы доступны через `strings rheolab.exe`. Злоумышленник может выпустить валидный токен и получить alpha/beta-манифесты до релиза. Сам updater подписан Tauri-ключом, поэтому это утечка сборок, а не RCE.
**Действие:**
- Ротация ключей на каждый major;
- и/или привязка токена к device-entitlement: сервер проверяет одновременно лицензию и устройство.

## P3 — nitpick

### 8. Дубликат `cfg`-кода в `is_e2e_mode` / `is_updater_disabled`
**Действие:** Унифицировать в helper `e2e_flag(name)` с двойным `cfg`. Сокращение ~15 строк.

### 9. `runCheck()` в `UpdateChecker.tsx` глушит ошибки
**Где:** строки 122–128.
**Проблема:** Фоновый тихий режим сбрасывает стор даже при ошибке; повторяющиеся сетевые сбои не видны в телеметрии.
**Действие:** Сэмплированный counter через `logger`/Sentry breadcrumb.

### 10. Капабилити `http:default` с allow `https://api.groq.com/*`
**Где:** `src-tauri/capabilities/default.json:60–63`.
**Действие:** Подтвердить, что вызов возможен только при user-provided API key и что ключ хранится зашифрованно (`encryption.ts`). Если Groq отключён в alpha — временно убрать allow до GA.

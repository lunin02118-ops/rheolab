Библиотека нужна для того, чтобы результат не оставался одноразовой выгрузкой из прибора. Здесь сохранённые эксперименты становятся рабочей базой: их можно искать, открывать повторно, сравнивать и использовать для отчётности.

## Что хранится в библиотеке

После сохранения в библиотеку эксперимент остаётся в локальной базе данных вместе с основными метаданными и результатами расчёта.

Обычно это позволяет:

*   вернуться к серии через день, неделю или месяц;
*   сравнить несколько экспериментов между собой;
*   быстро подготовить повторный отчёт;
*   не держать рабочую историю в папках, почте и временных файлах.

## Как устроен раздел

В библиотеке есть два рабочих блока:

*   **Эксперименты** — сохранённые результаты испытаний;
*   **Реагенты** — отдельный каталог реагентов для внутренней работы лаборатории.

Для списка экспериментов доступны два режима просмотра:

*   список;
*   плитка.

## Поиск и фильтрация

В обычной работе библиотека используется как рабочий архив. Поэтому здесь важны не только сами записи, но и быстрый отбор нужной серии.

Обычно в библиотеке:

*   ищут нужный эксперимент;
*   фильтруют результаты по основным признакам;
*   открывают старую серию повторно;
*   переходят из сохранённого результата в сравнение или в отчёт.

## Каталог реагентов

Во второй вкладке библиотеки находится каталог реагентов. Он нужен для внутренней систематизации лабораторной химии.

В каталоге можно:

*   искать реагенты по названию и производителю;
*   группировать их по категориям;
*   хранить производителя, страну и форму реагента;
*   использовать единый справочник в повседневной работе.

## Когда библиотека особенно полезна

Библиотека становится особенно важной, когда лаборатория работает не с одним тестом, а с повторяющимися сериями:

*   подбор рецептур;
*   сравнение партий;
*   архивирование контрольных испытаний;
*   подготовка повторных отчётов без повторного импорта исходного файла.

Это короткое руководство поможет установить RheoLab Enterprise, запустить пробный режим и пройти путь от первого файла до первого сохранённого результата.

## 1. Установка

1.  Скачайте установочный файл для Windows с [главной страницы](/) или через [прямую ссылку](/download/latest).
2.  Запустите `RheoLab-Enterprise-Setup.exe`.
3.  Следуйте инструкциям мастера установки.
4.  После завершения установки приложение запустится автоматически.

> Рекомендуем устанавливать программу на рабочее место, где будут храниться ваши локальные данные и библиотека результатов.

## 2. Первый запуск

При первом запуске вам будет предложено активировать пробный режим или ввести лицензионный ключ.

*   Выберите **"Начать пробный период"**, чтобы проверить основной рабочий сценарий на ваших реальных файлах.
*   Если у вас есть ключ, введите его в соответствующее поле.
*   Если вы оцениваете продукт для команды, начните с реальных файлов вашей лаборатории.
*   В пробном режиме достаточно пройти полный цикл: открыть файл, сохранить результат, сравнить его с другим опытом и подготовить отчёт.

## 3. Откройте первый файл

1.  На странице **Анализ** нажмите кнопку загрузки файла или просто перетащите файл в окно программы.
2.  Обычно используются форматы `.xls`, `.xlsx`, `.csv`, `.txt`, `.dat`.
3.  После загрузки вы увидите график, таблицу исходных данных, рабочие вкладки результата и расчёты по циклам.
4.  При необходимости дополните результат рецептурой, анализом воды, данными по лаборатории и другими рабочими полями.

## 4. Сохраните результат в библиотеку

После первого открытия файл не должен оставаться одноразовой выгрузкой.

1.  Проверьте график и данные на экране.
2.  Сохраните результат в библиотеку, чтобы он не потерялся после закрытия файла.
3.  При сохранении можно указать оператора и лабораторию, если вы ведёте внутренний учёт.
4.  После этого серию можно будет снова найти, открыть, сравнить и выгрузить в отчёт.

## 5. Что делать дальше

После первого файла обычно переходят к одному из трёх сценариев:

1.  Открыть [импорт данных](/docs/parsing), если вы работаете со смешанным парком приборов.
2.  Перейти в [библиотеку](/docs/library), если хотите наладить хранение и повторное открытие серий.
3.  Открыть [сравнение](/docs/comparison), если вам важно сопоставлять несколько экспериментов на одних графиках.
4.  Открыть [частые вопросы](/docs/faq), если вы оцениваете пробную версию и хотите понять лицензии, хранение и сетевые требования.

Этого обычно достаточно, чтобы понять главный сценарий программы: открыть файл, проверить расчёты, сохранить результат в библиотеку и вернуться к нему позже.
